Miloslav Beno
t miloslavbeno
Miloslav is co-founder of devsense where he is working on phalanger, the php compiler for .net/mono For more info head to his linkedin profile.
Miloslav will talk about:
  • Phalanger – The PHP compiler for .NET
Phalanger compiles PHP code into .NET which can then be executed natively on the server (in 32 or 64bit), just like C# code. By compiling the PHP code into .NET assemblies, not only is the code significantly faster than native PHP but once it is compiled into .NET it can be shared easily with C# code or other .NET language. Other benefits include possibility to use modern IDE as Visual Studio and other productivity tools available for .NET developers. Compiling PHP code also enables possibility of source-less and obfuscated distribution.

In this talk, I'll present three most appealing use cases of Phalanger. First, we look at performance. We’ll compare benchmarks of PHP 5.4 and Phalanger and explore how this affects real world PHP application e.g WordPress. Second, we look how PHP can benefit from close ASP.NET integration, as being able to simply use IIS output caching in kernel mode or integrate users through membership providers with existing ASP.NET application. Third, we look how Phalanger simplifies integration between open-source PHP applications and .NET world on the example of C# plugin for WordPress.

Mariusz Gil
t mariuszgil
Programmer and software architect connected with industry more than 12 years. He is currently designing and implementing web application architectures in project where the capacity factor is crutial. Previously worked as architect, tech-lead, and developer in "nasza-klasa" and "gadu-gadu" (called now "gg networks").
Mariusz will be speaking two times on webclusters with 2 awesome topics
First topic:
  • Varnish’em all
Varnish is modern web applictions accelerator and many times it's described as caching reverse HTTP proxy. Placed in the infrastructure right before http servers relieves them by caching generated content, while maintaning a very high performance. Varnish can do much more than just a simple caching of content. Configuration language VCL allows you to change the behavior of server also allows for load balancing with control of backends. Partial support for Edge Side Includes or URL rewriting are some of the possibilities by which Varnish is used in services like Facebook, Wikia or Polish service gazeta.pl

During the session we will trace installation, launch and configuration process of Varnish in a typical web application with big/small infrastructure. During the presentation we will also meet with VMODs which are helping with programming advanced logic in VCL.
Second topic:
  • Other aspects of memcached and MySQL.
Memcached and MySQL are solutions, which probably needs no introduction to anyone. They are working great standalone but together they can solve most of the performance problems (mainly with database). It should be noted that both projects complement each other perfectly well in other situations which are far from typical caching of data.

In the first part of the presentation we will meet HandlerSocket which introduces simplified and directed access to database storage-table engine, which can result in increase in reading data base on primary key up to several hundred thousands per second. HandlerSocket became the prototype introduced in MySQL 5.6 memcached API to which we will devote more attention.

The second part will be focused on the use of MySQL to transferr data between memcached clusters in different locations. In the case of scaling applications in separate locations there is often need to place in each of them local, consistent copy of data cache to provide fast and easy access to key data. On the example of Facebook architecture we will look at the problems of fully resistant method to solve this problem, which in addition can also be used in smaller projects, such as session data redundancy.
